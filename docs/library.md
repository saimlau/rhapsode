# Library GUI

```bash
./rhapsode --gui
```

starts a local web app (bound to 127.0.0.1 only) and opens the browser.
The library — imported PDFs, generated read-along bundles, and
`library.json` — lives at `[library] path` from `config.toml`.

## Adding papers

Drag PDFs anywhere onto the page. Each paper is ingested (a fast metadata
pass names the card immediately), queued, and generated by a single worker
that keeps the model warm between papers. Progress is live: synthesis
accounts for nearly all of the bar; page rendering and the manifest land in
the final few percent. Re-adding a PDF you already have is detected by
content hash — it won't duplicate.

## Listening

Click a card to open the paper's read-along view in place. Your position in
every paper is remembered server-side, and papers auto-advance
podcast-style when one ends (toggleable). Drag cards to reorder; hover a
card for regenerate/remove.

## Playlists

A paper can be in several playlists; its audio exists once. The sidebar
header switches between **All papers** and named playlists. Add papers via
the ＋ on a row or the right-click menu; within a playlist, ✕ removes from
the playlist only (the paper stays in the library). Reorder and
auto-advance follow the active playlist. Names containing `" / "` render as
indented subplaylists under their parent — this is how the
[Zotero plugin](zotero.md) mirrors collection trees.

## Multi-select & context menu

++shift++-click selects a range, ++ctrl++-click toggles items. Right-click
for **Add to playlist**, **Remove**, **Regenerate**, and **Sort by** —
title, year, or recently added; picking the same sort again flips its
direction.

## Server lifecycle

The worker idles politely: the Kokoro model parks to CPU RAM and later
unloads entirely ([`[tts]` lifecycle keys](configuration.md)), and a
Rhapsode-started GROBID service stops after its own idle window. With
`[gui] idle_exit_min` set, the whole server exits after that many minutes
without real activity (uploads, playback, page loads count; idle background
tabs don't) — pair it with `packaging/rhapsode.service` so the next use
starts it again on demand. `GET /api/status` reports what's currently
resident.

## REST endpoints

For scripting, the server speaks plain JSON: `GET /api/library`,
`POST /api/papers` (multipart upload), `POST /api/papers/by-path`,
playlist CRUD under `/api/playlists…`, `PUT /api/queue` (order/auto-advance),
`POST /api/papers/{id}/position`, `POST /api/papers/{id}/regenerate`,
`GET /api/status`, and a server-sent-events stream at `/api/events`.
`POST /tts` synthesizes ad-hoc text (this is what the
[speechd voice](system-voice.md) uses).
